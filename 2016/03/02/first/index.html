成功配置git page + hexo

2016年03月02日

前言

经过漫长努力，终于配置好git page + hexo的博客了，算是有了一个自己的博客了。当然摸索和查资料了好久，也踩了很多坑，也有很多东西没有做。后面再继续说~

简要过程

  1. 配置git page
  2. 本地配置hexo
  3. hexo和git page 关联
  4. 主题的修改

详细内容

配置git page

这一块没什么好说的，基本上按照教程上来就行。这个弄完，直接可以访问，相当于多了一个免费的博客地址。地址一般为：name.github.io

比如我的:

liwenlong.github.io

本地配置hexo

这个也非常简单,搜索相关教程即可,弄完之后，就可以本地启动个服务器，非常简单。
注意 这里配置hexo一定不要在刚才git page 的本地目录里面进行。它初始化会删除掉.git

hexo和git page 关联

其实hexo和git page是两个东西,hexo是一个轻量的静态博客系统。利用node,可以本地启动静态服务器，也可以将你写的东西进行编译。实际上展示的就是

主题+文章

所以,想要把hexo 和 git page 关联起来，需要3步:

  1. hexo里面安装一个插件
    npm install hexo-deployer-git --save

  2. hexo配置_config.yml文件的deploy配置
    这里格式要求比较严，注意空格和格式

    deploy: type: github repo: git@github.com:yourname/yourname.github.io.git$ branch: master

  3. hexo deploy
    执行正常情况，会让你输入push到github 上面的用户名和密码
